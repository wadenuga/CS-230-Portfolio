# CS-230-Portfolio
## The Gaming Room Client 

The Gaming Room wanted to expand its game, Draw It or Lose It, from Android to multiple platforms. The goal was to design software that would support web, desktop, and mobile environments while allowing multiple users to play together. The software needed to be scalable, secure, and easy to maintain. 

## What I Did Well 

I did well evaluate different operating platforms and compare their strengths and weaknesses. I also explained why Java and object-oriented programming were good choices for building a cross-platform application. 

## Helpful Parts of the Design Process 

Creating the software design document before writing code helped organize the project. It made it easier to identify the requirements, choose an appropriate architecture, and avoid potential design problems before implementation. 

## What I Would Improve 

If I could revise one part of the project, I would expand the system architecture section by adding more diagrams and providing additional details about security and cloud deployment. This would make the design easier for developers and stakeholders to understand. 

## Understanding the User’s Needs 

I focused on the client’s requirements by designing software that supports multiple platforms, provides good performance, and allows many users to access the game simultaneously. Considering the user’s needs is important because it helps ensure the final product is reliable, easy to use, and meets business goals. 

## My Design Approach 

I approached the project by first analyzing the requirements, then comparing platform options, and finally selecting technologies that best meet the client’s needs. In future projects, I will continue using requirement analysis, UML diagrams, design patterns, and early planning to create efficient and maintainable software.
